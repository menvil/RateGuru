# Code of Conduct

RateGuru contributors are expected to keep project communication respectful,
focused, and constructive.

## Expected Behavior

- Be clear and professional.
- Assume good intent while verifying technical claims.
- Keep feedback focused on code, documentation, and project outcomes.
- Respect project scope and established workflow rules.

## Unacceptable Behavior

- Harassment, threats, or personal attacks.
- Discriminatory language or conduct.
- Public disclosure of private security reports before coordinated disclosure.

Maintainers may remove comments, close issues, or restrict participation when
behavior disrupts the project.
